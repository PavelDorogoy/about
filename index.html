Опыт работы

CTO стартапа / Epiczaim
Сентябрь 2020 - Сейчас

Руководил разработкой в стартапе «Epiczaim», который занимается оказанием рекламных услуг для микрофинансовых организаций. Разработал проект, архитектуру сервиса с нуля. Использовался стек:

Linux, SSH, Cron, Git, Docker, RabbitMQ, NGINX, Node JS, MySQL, Cloudflare, PHP, Composer, Symfony, TWIG, Yarn, Webpack, SASS, Vue JS, JQuery.

В основе была слоистая архитектура с вертикальным делением на контексты бизнеслогики. Применялось строгое разделение back-end и front-end. Они взаимодействовали между собой через REST API. Для автодокументирования API использовался Swagger. Для асинхронного выполнения долгих однотипных команд использовался Symfony Messenger в связке с RabbitMQ. Активно использовался EventDispatcher для создания универсальных бандлов (модулей).

Сервис является высоконагруженным, каждый переход учитывается, анализируется, определяются поведенческие факторы клиентов. Каждый день происходит более 10 000 переход.

За это время было реализовано:

  • Разработка и реализация всей архитектуры сервиса.
  • Модульная административная панель - бандл, в который можно добавлять категории, подкатегории, страницы с разными уровнями доступа. На каждой странице можно добавлять различные компоненты, конфигурировать их. Было разработано больше 15 компонентов (например таблицы, формы, графики и т.д.).
  • Модуль защиты от DDoS атак - использовался iptables и интеграция с сервисом Cloudflare.
  • Бандл для работы с пользователями (группы, роли, права).
  • Бандл безопасности (способы входа, сессии, режим "Запомни меня").
  • Бандл кэширования - с использованием слоев кеша и фабрикой провайдеров кеша.
  • Бандл управления контентом - управление содержимым сайта через административную панель. Узлы контента настраиваются в виде дерева.
  • Бандл для работы с файлами - с возможностью загружать и скачивать файлы в разные папки с проверкой прав пользователя, форматов и размера файлов. Была реализована возможность удалять неиспользуемый файлы из папок. Была добавлена страница в административной панели для работы с файлами и папками.
  • Бандл для генерации ссылок и отслеживания трафика - создаются ссылки, которые осуществляют редирект на разные адреса и отслеживаются все переходы (ip, http_referrer, query). Также есть возможность генерации таких ссылок в виде графического пикселя.
  • Бандл email и sms рассылки.
  • Бандл с фабрикой telegram-ботов.
  • Бандл Партнерских сетей - фабрика партнерских сетей (организаций, предоставляющих трафик на различные ресурсы). Была произведена интеграция с 5 партнерскими сетями.
  • Бандл "Планировщик" - в первом приближении аналог crone, управляемый через административную панель. Осуществляет выполнение кода по расписанию с точностью до секунды. Расписание создается на основе шаблонов (интервалов времени с заданной частотой) и может быть составным. Например, с Пн по Пт одна частота, с Сб по Вс другая вплоть до часов и минут. Интервалы могут быть дискретными или распределенными случайно.
  • Система управления клиентами - система управления, которая позволяет находить клиентов, помогать менять им логин/пароль, выводить информацию по транзакциям платной подписки, отписывать клиентов от платной подписки и email и sms рассылок.
  • Система управления вебмастерами - теми, кто приводит трафик. Была реализована большая система анализа качества трафика с дашбордами, фильтрами, графиками, дневными отчетами.
  • Система управления платными подписками.
  • Платежный бандл - универсальный бандл с фабрикой и конфигурированием платежных систем (была произведена интеграция с 5 платежными системами) и фабрикой фискальных сервисов (для формирования чеков).
